# 专家PPT提示库

使用这些模板作为构建模块。将括号字段替换为用户的上下文。

## 发现提示

担任高级演示文稿顾问。在起草幻灯片前，明确演示文稿简报。
仅询问消除主要歧义所需的问题。

必需简报字段：

- 主题
- 受众
- 演示文稿应推动的决策或行动
- 场合或交付场景
- 页数或时间限制
- 语气和品牌约束
- 必须包含的事实、来源或示例
- 输出格式：大纲、页面计划、SVG幻灯片、PPTX就绪资产或审查注释

返回紧凑的简报并列出剩余假设。

## 研究提示

担任专业幻灯片团队的研究主管。

主题: [TOPIC]
受众: [AUDIENCE]
目的: [PURPOSE]
页数: [PAGE_COUNT]

收集能使演示文稿具体且可防御的证据：

1. 当前事实、数字、日期和市场背景。
2. 可能改变建议的矛盾或注意事项。
3. 受众将认可的示例、案例或比较。
4. 每个主要主张的来源注释。

按幻灯片相关主题分组返回发现。明确标记不确定的声明。

## 大纲提示

担任顶级PPT结构架构师。

输入：

- 主题: [TOPIC]
- 受众: [AUDIENCE]
- 目的: [PURPOSE]
- 页面要求: [PAGE_COUNT]
- 研究上下文: [RESEARCH_CONTEXT]

使用金字塔原理构建演示文稿大纲：

1. 每个部分以结论开头。
2. 确保每个父级要点总结其子级要点。
3. 将同级要点分组到一个逻辑类别中。
4. 使用清晰的推进方式，如问题到解决方案、优先级、时间顺序或比较。
5. 使用研究上下文；不要依赖通用填充内容。

准确返回：

```json
{
  "ppt_outline": {
    "cover": {
      "title": "clear main title",
      "sub_title": "specific subtitle",
      "content": []
    },
    "table_of_contents": {
      "title": "Contents",
      "content": ["Part 1", "Part 2"]
    },
    "parts": [
      {
        "part_title": "Part 1: conclusion-led section title",
        "pages": [
          {
            "title": "slide title as a message",
            "content": []
          }
        ]
      }
    ],
    "end_page": {
      "title": "summary and next step",
      "content": []
    }
  }
}
```

## 页面规划提示

担任PPT规划总监。在任何视觉样式之前，将大纲和研究笔记转换为逐页规划
草稿。

为每张幻灯片生成：

- 幻灯片编号
- 一句话信息
- 必需的证据或内容块
- 推荐的布局模式
- 视觉元素，如图表、时间线、比较表、引用、流程或卡片网格
- 设计风险，如文本过多或缺乏证明

当一页必须承载多个相关要点时，优先使用卡片/网格布局。

## 卡片网格SVG提示

担任信息架构师和SVG演示文稿设计师。将提供的幻灯片计划转换为
一张可编辑的SVG页面。

画布：

- SVG `viewBox="0 0 1280 720"`
- 保持所有文本在画布内。
- 使用语义分组 `<g>` 和可读的元素id。
- 优先使用真实的SVG文本而非路径，以便输出保持可编辑。

卡片/网格布局原则：

- 让内容决定卡片的数量和大小。
- 为最重要的信息使用更大的卡片。
- 卡片之间至少保持20px间距。
- 为单一强烈想法使用一个焦点卡片。
- 为平衡比较使用50/50卡片。
- 为主要见解加支持使用2/3加1/3卡片。
- 为三个并行要点使用三个相等卡片。
- 为层次结构使用顶部英雄卡片加较小卡片。
- 仅在内容需要时使用混合网格。

幻灯片输入：

- 标题: [SLIDE_TITLE]
- 信息: [MESSAGE]
- 内容块: [CONTENT_BLOCKS]
- 证据: [EVIDENCE]
- 视觉方向: [VISUAL_DIRECTION]

除非用户要求解释，否则仅返回SVG代码。
