# 专家PPT工作流

使用此技能将演示文稿主题转化为专业的幻灯片制作工作流：
需求发现、证据收集、金字塔原理大纲、页面规划，然后是可编辑的
SVG幻灯片草稿。

## 核心规则

不要从主题直接跳转到完成的幻灯片。将用户视为导演，充当小型专家
团队：顾问、研究员、结构架构师、页面规划和视觉设计师。

## 工作流

1. 在起草前明确意图。
   仅询问缺失的高影响力问题：受众、目标、场合、语气、页数、来源
   约束、必须包含的要点和期望的输出格式。

2. 在确定大纲前收集证据。
   当主题依赖最近的事实、市场、工具、
   公司、政策、定价或技术状态时，使用当前、有来源支持的信息。在工作
   材料中保留引用或来源注释。

3. 使用金字塔原理构建大纲。
   结论先行，归类相关要点，使每个部分总结其子级，并按
   清晰的逻辑（如问题到解决方案、时间顺序、优先级或比较）排序各部分。

4. 创建逐页规划草稿。
   为每张幻灯片指定信息、证据、布局意图、视觉元素和演讲者价值。
   这是从"内容策略"到"视觉设计"的交接；不要跳过它。

5. 当用户需要可编辑页面时，生成设计草稿为SVG。
   使用1280x720画布和卡片/网格布局。使布局由内容驱动，最
   重要的想法获得最大的视觉区域。

6. 在可审查的检查点呈现结果。
   推荐：发现摘要 -> 研究大纲 -> 页面计划 -> SVG提示/输出。仅在
   需要用户判断时请求批准，例如更改受众、故事角度或视觉
   方向。

## 辅助脚本

使用辅助工具为每个工作流阶段生成可复用的提示包：

```bash
bash /mnt/skills/user/expert-ppt-workflow/scripts/ppt-workflow.sh <phase> [topic] [page_count] [audience] [purpose]
```

阶段：

- `discovery` - 顾问问题和简报清单
- `research` - 证据收集简报
- `outline` - 金字塔原理大纲提示
- `page-plan` - 幻灯片规划提示
- `svg` - 使用卡片/网格布局的可编辑SVG幻灯片提示
- `all` - 所有提示包合并为一个JSON对象

脚本将状态消息写入stderr，将JSON写入stdout。

## 输出模式

对于新演示文稿，返回：

1. `简报` - 受众、目标、故事角度、页数、约束。
2. `证据计划` - 搜索什么、哪些事实重要，以及来源注释。
3. `PPT大纲` - 封面、目录、部分、页面标题和结束页。
4. `页面计划` - 每页一行：信息、内容、布局、视觉处理。
5. `SVG草稿` - 每张幻灯片一个SVG文件或代码块（如请求）。

对于改进现有演示文稿，返回：

1. 逻辑、证据和布局问题的诊断。
2. 修订的大纲或页面顺序。
3. 页面级重写计划。
4. 需要重建的页面的SVG或设计提示。

## 质量标准

- 每张幻灯片必须有一个主要信息。
- 证据必须支持主张；不要编造统计数据。
- 布局必须服务于内容层次结构，而不是装饰性模板。
- 卡片间距应保持一致；在SVG草稿中卡片之间至少保持20px间距。
- SVG viewBox 必须是 `0 0 1280 720`。
- 使文本适应其视觉容器；在缩小到不可读大小之前，拆分长文本或减少内容。

## 故障排除

如果用户只提供模糊的主题，从 `discovery` 开始。
如果主题具有时效性，在 `outline` 之前进行研究。
如果演示文稿感觉泛泛而谈，添加更多证据并 sharpen 受众特定的结论。
如果SVG杂乱，减少幻灯片内容或将其拆分为多个页面。
